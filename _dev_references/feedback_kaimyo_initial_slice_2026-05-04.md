## 観測された摩擦点（5 件）

### F1：素材薄さ（最重要）

- 14 motifs 中、初段クエリ（`category:景気導入` AND `文体:対句`）にヒットするのは **m01 1 件のみ**
- 住職フィードバック「五段それぞれの表現パターンが少なく毎回似た内容に思える」と完全一致
- **対応**：性霊集 idx=72 以外（特に序系・無常素材系）からの抽出を急ぐ。`shoryoshu_genre_index.md` の「序(8)」「無常素材(11)」「達嚫(5)」「弔辞追悼(28)」を優先候補に

### F2：文体ミスマッチ（gendaigoyaku は諷誦文に直接使えない・最重要）

- §10 仕様は「text_gendaigoyaku を再構築」と指示
- しかし出力 1 と出力 2 を並べると、**gendaigoyaku 再構築版（出力 1）は「である」「変転する」など現代口語が混ざり、諷誦文の文語体に合わない**
- 一方 **kakikudashi 版（出力 2）は「凝然たり」「起滅す」など文語締めで、jitsurei の実例とも整合する**
- buntai ルール「二段では『〜であった』を並べず…文語体を使うこと」は実質的に**全段に適用される**
- **設計判断要請**：諷誦文本体は kakikudashi を直接使い、gendaigoyaku は学習者向けの参照（kaimyo-app の補助表示）に位置づけるのが正しい
- §10 を「kakikudashi をベースに、固有名詞のみ profile で再構築」に修正提案

### F3：補注の混入

- m01 の gendaigoyaku は補注（括弧書き）を **7 箇所**含む
- 全削除すると意味が削がれる（例：「寥廓（りょうかく・広漠として広大な）たる」→「寥廓たる」で読み手は寥廓の意味を失う）
- 全残置すると諷誦文として読み下せない
- **対応案**：補注を別フィールド（`text_gendaigoyaku_clean` + `annotations[]`）に分離。kaimyo-app 側で表示と本体生成を分けて使う

### F4：season タグ不在

- 14 motifs すべてに `season:*` タグなし
- preferred_tags での季節合致スコアリングが no-op
- **対応**：motif 抽出時に没月マッチを意識した季節タグ付与を必ず行う（例：m01「寥廓」は通年的なので `season:通年` 付与）

### F5：硬さタグ不在

- 14 motifs すべてに `硬さ:*` タグなし
- kaimyo-app 側に既に「柔/中/硬」の UI が実装されている（commit `d79d5a1`）のに、motif 側に対応軸がないため繋がらない
- 設計メモ §8 にも「硬さ軸の倉庫原則上の妥当性再評価」が残課題として記載
- **対応案**：(a) motif に `硬さ:*` を付与、または (b) `motifs_index_design.md` の方針通り「漢語密度／対句密度／典故密度」に分解して付与

## 追加発見：初段冒頭句のミスマッチ

## 2026-05-04 追記：F2 の解像度向上（ケンシン判断）

### 経緯

縦割り完走後、ケンシンが出力 1（gendaigoyaku 再構築版）と出力 2（kakikudashi 版）を見比べて：

> 書き下しそのままではだめですね。
> でも大師の息遣いは聞こえてくる響きをもった文章ですね。
> 現代語でうまく表現できないものか。

これを受けて 3 段階の試訳を提示：

- 試案 A（雅文寄り・「ものなり」「消えゆく」型）
- 試案 B（中庸・助動詞は現代寄り、語彙は重み残す）
- 試案 C（柔らか・完全に現代口語）

ケンシン判断：**試案 A を採用**。

### 確定した設計判断

**諷誦文の現代語訳の標準位置は「雅文体（がぶんたい）」**。

- 学習者向けの説明訳（現状の gendaigoyaku）でもなく
- 文語そのまま（kakikudashi）でもなく
- 現代日本語の文法で書かれているが、雅文の重みと息遣いを保った文体
- 例（m01 の試案 A）：

```
そもそも、ひろやかなる法身の本体は、もろもろの因縁を超えて、しずかに動かぬものなり。
されど、煩悩に染められたる心の海は、縁にしたがいて起こり、また消えゆく。
```

### 倉庫側（buddhist-data-api）への提案

motifs.json schema に**雅文体フィールドを追加**する：

```json
{
  "id": "m01",
  "text_kakikudashi": "夫れ寥廓たる性虚は…",       // 既存・原典そのまま
  "text_gendaigoyaku": "そもそも寥廓（りょうかく…",  // 既存・学習者向け説明訳
  "text_gendai_gabun": "そもそも、ひろやかなる…",   // 新規・諷誦文本体に直接使う雅文体
  "tags": [...]
}
```

**役割再定義**：

| フィールド | 役割 | 諷誦文本体での使用 |
|---|---|---|
| `text_kakikudashi` | 原典の書き下し | 「硬」モード時 |
| `text_gendai_gabun` | **雅文体訳（新）** | **「中」モード時（標準）** |
| `text_gendaigoyaku` | 学習者向け説明訳（補注付き） | 使わない（kaimyo-app の補助表示や別用途） |

### kaimyo-app 側の既存「硬さ」UI との接続

既に実装済の 3 段階 UI（commit `d79d5a1`：柔/中/硬）と直結：

| 硬さ UI | 使用フィールド |
|---|---|
| 硬 | `text_kakikudashi` |
| 中（標準） | `text_gendai_gabun` ← 新規 |
| 柔 | （将来）`text_gendai_kuchigo` 等のさらに柔らかい訳 |

これで F4・F5（season・硬さタグが motif にない）の「硬さ」側も**構造的に解消**する（タグではなくフィールド分離で表現）。
