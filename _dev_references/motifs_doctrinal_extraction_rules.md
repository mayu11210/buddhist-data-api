# 教義書 9 著作からの大師の言葉 motif 抽出 設計書（2026-05-07 初版）

## 0. 本書の位置付け

本書は kaimyo-app 側「諷誦文 段階1（橋スロット候補拡張）」（2026-05-07 commit `82cc61d`）の続編として、**段階2「教義書 9 著作からの大師の言葉 motif 抽出」** の設計を確定するために作成する。

`_dev_references/motifs_index_design.md`（2026-05-04 初版）の倉庫原則・タグ軸体系を継承する。本書は教義書系 motif 化の **追加ルール・タグ判定基準・kaimyo-app 側対応要件** に特化する。

データ追加（motifs.json への新規 motif 反映）は性霊集 motif 化の並行作業（候補 B 第 4 ラウンド継続中）が完走した後に開始する。本書はそれまでの **設計の凍結文書** として機能する。

## 1. 段階1 の達成状況と段階2 の必要性

### 段階1 の実態（2026-05-07 commit 82cc61d 後）

`lib/fusonmon-orderer.ts` の `orderForBridgeStep` で 3 階層構造を実装：

| 候補プール | 件数 | 出所 | スコア |
|---|---|---|---|
| (1) `daishi_kotoba` | 5 | fusonmon.json 直書き | base 4〜5 |
| (2) `引用形式:大師曰く` motif | 2 | motifs.json | base 4〜5 |
| (3) 性霊集系（段階1 拡張） | 153 | motifs.json `出典:性霊集_idx*` | base 1 + persona +2 + category +1〜2 |
| **合計** | **160** | | |

性霊集系 153 件の category 分布：讃美 74 / 悲嘆 38 / 無常 34 / 廻向 29。

### 段階2 が必要な理由

性霊集系 153 件は **「弔辞・願文・碑文の文学性」** が中心の素材。一方、教義書 9 著作には **「教義的・哲学的な大師の言葉」** が眠っており、性霊集にはない色を加える価値がある：

- 即身成仏義の「六大無礙にして常に瑜伽なり」「三密加持すれば速疾に顕わる」
- 般若心経秘鍵の「真如外に非ず、身を棄てて何くんか求めん」
- 秘蔵宝鑰の「生まれ生まれ生まれ生まれて、生の始めに暗く、死に死に死に死んで、死の終りに冥し」
- 三教指帰の「天、朗かなるときは則ち象を垂れ、人、感ずるときは則ち筆を含む」

これらは諷誦文の橋スロット（大師の言葉スロット）に入れた時、性霊集の文学的弔辞とは **異質な厳粛さ・教義的重み** をもたらす。

### 段階2 完了時の規模見込み

各著作 5〜10 セッションで **265〜440 件**追加見込み。性霊集系の同時並行進行（候補 B 残り 18 篇 ≒ +126 件）と合算すると、最終 motif 総数は **約 500〜600 件規模**。

## 2. 倉庫原則の再確認（最重要）

`motifs_index_design.md` §1 で確定済の原則を本書でも遵守する：

### 派生原則：アプリ用途で命名しない

`用途:諷誦文橋` `用途:法話引用` のような **アプリ固有のタグ軸を新設してはならない**。これらは kaimyo-app 側が `category:*` `文体:*` `典故:*` 等の自然なタグ組合せから判断する。

倉庫側でやるのは：
- 句の自然な属性（出典・典故・文体・密教教学要素・主題・含意）を網羅的にタグ付け

倉庫側でやらないのは：
- 「この句は橋に向く」「この句は法話に向く」のような用途判断
- 「諷誦文用」「法話用」のようなアプリ命名

### 派生原則：用途判定は利用側に閉じる

kaimyo-app 側 `fusonmon-orderer.ts` の `orderForBridgeStep` がやっていること：
- `出典:性霊集_idx*` AND `category:無常 / 悲嘆 / 廻向 / 讃美` AND `文体:対句 / 短句` でフィルタ

これと同じ要領で、教義書系も kaimyo-app 側で：
- `典故:即身成仏義` OR `典故:吽字義` OR ... AND `category:密教教学 / 無常 / 景気導入` AND `文体:対句 / 短句` でフィルタ

倉庫側はこの判定に必要な自然タグを **網羅的に正確に** 付ければよい。

## 4. 抽出ルール

### 4.1 1 motif の単位

- **句長**：原則 8〜80 字（性霊集 motif と同じ範囲）
- **完結性**：句点で区切った時に意味が独立して通じること
- **対句・畳語**：対句構造は分割せず 1 motif として保持（例：「生まれ生まれ生まれ生まれて、生の始めに暗く、死に死に死に死んで、死の終りに冥し」は 1 motif）

### 4.2 「引用形式:大師曰く」タグの適用判定（最重要）

| 著作 | 著者 | 引用形式タグ | 理由 |
|---|---|---|---|
| 即身成仏義 | 空海 | `引用形式:大師曰く` | 空海自身の著作 |
| 吽字義 | 空海 | `引用形式:大師曰く` | 同上 |
| 声字実相義 | 空海 | `引用形式:大師曰く` | 同上 |
| 弁顕密二教論 | 空海 | `引用形式:大師曰く` | 同上 |
| 般若心経秘鍵 | 空海 | `引用形式:大師曰く` | 同上 |
| 秘蔵宝鑰 | 空海 | `引用形式:大師曰く` | 同上 |
| 三教指帰 | 空海 | `引用形式:大師曰く` | 同上（青年期著作） |
| 菩提心論 | 不空訳（龍樹著） | **不付与** | 訳経・空海の言葉ではない |
| 大日経疏巻第一 | 一行記 | **不付与** | 一行阿闍梨の記・大日経注釈 |

ただし例外として、**菩提心論・大日経疏巻第一の中で空海が独自解釈や引用解説を加えている部分**は、その部分のみ `引用形式:大師曰く` を付与する。区別は kakikudashi 中の文脈で判断（既に kukai/*.json に取り込まれている時点で「空海関係著作」として位置付けられているが、文体・主体が空海ではない部分が大半）。

実務的には：
- 菩提心論：原則「不付与」。空海加筆部分が確認できた場合のみ付与
- 大日経疏巻第一：原則「不付与」。`典故:大日経` `典故:大日経疏` のみで運用

### 4.3 タグ軸の選択（既存 9 標準軸 + 6 特殊軸の運用）

#### 必須軸

- `出典:<著作キー>`（例：`出典:即身成仏義` `出典:秘蔵宝鑰`）
- `category:*`（後述カテゴリ表）
- `文体:*`（対句 / 長句 / 短句 / 畳語 / 散文 / 呼びかけ 等）

#### 推奨軸

- `典故:<著作名>`（自著作なら自典故・他典故引用なら経典名 例：`典故:大日経` `典故:法華経`）
- `密教:<教学概念>`（六大 / 三密 / 即身成仏 / 阿字 / 本不生 / 五智 / 加持 / 曼荼羅 / 法身説法 等）
- `主題:*`（菩提心 / 無常 / 生死 / 本来性 / 智慧 / 求道 等）

#### オプション軸

- `引用形式:大師曰く`（§4.2 の判定に従う）
- `成句:famous`（漢文成句のまま儀礼朗誦されるレベルの著名句）
- `一句性:核心 / 補助 / 導入`（人生集約句として単独成立するかの評価）
- `含意の射程:全人生 / 晩年 / 臨終 / 転機`

#### category 軸の値（教義書での出現傾向）

| category 値 | 想定出現 | 例 |
|---|---|---|
| `category:密教教学` | 多 | 即身成仏義・吽字義・声字実相義・二教論の大半 |
| `category:無常` | 中 | 秘蔵宝鑰序文・三教指帰序章 |
| `category:景気導入` | 少〜中 | 各著作の冒頭・序文 |
| `category:典故引用` | 中 | 二教論・大日経疏巻第一の経典引用部分 |
| `category:讃美` | 少 | 仏徳讃美部分 |
| `category:呼びかけ` | 少 | 三教指帰の対話部分 |
| `category:発願` | 少 | 各著作の結句 |

注：性霊集系で多い `category:悲嘆` `category:廻向` `category:人柄描写` は教義書ではほぼ出ない。これは正しく分布の違いとして反映する。

### 4.4 source フィールドの構造

```json
"source": {
  "corpus": "kukai_doctrinal",
  "work": "sokushin-jobutsu",
  "work_ja": "即身成仏義",
  "section": "三 六大無礙常瑜伽",
  "T_number": "T2428",
  "vol": 77
}
```

`corpus` は新規値 `kukai_doctrinal` を導入（性霊集系 `shoryoshu_miyasaka` と区別）。`work` キーは kukai/*.json のファイル名と一致させる（即身成仏義 = `sokushin-jobutsu`）。

`section` は kakikudashi 内の章節見出し（取れる場合のみ）。取れなければ省略可。

### 4.5 text_gendai_gabun（雅文体訳）の手当て

教義書系 motif の初期段階では **`text_gendai_gabun` は省略可** とする：

- 橋スロットでは `text_kakikudashi` がそのまま「大師曰く」引用に流れる
- kaimyo-app 側の renderer は `m.text_gendai_gabun ?? m.text_kakikudashi` のフォールバックを既に実装済（`fusonmon-orderer.ts` 1387-1388 行）
- 雅文体訳は将来「二段以降にも教義書 motif を使いたい」要件が出たら後追いで足す

`text_gendaigoyaku`（口語訳＋典故注記）は **必須**。kakikudashi の文脈解釈・出典注記のため。教義書 kukai/*.json の document レベル gendaigoyaku から該当部分を抜き出して motif 単位に整形する。

## 5. kaimyo-app 側の対応要件

### 5.1 橋スロットフィルタの拡張（要実装）

現状の `orderForBridgeStep` は性霊集系のみを対象とする：

```ts
if (!isSeireishuMotif(m)) continue;  // 行 728
```

これを教義書系も含むよう拡張：

```ts
const isDoctrinal = m.source.corpus === 'kukai_doctrinal';
const isSeireishu = isSeireishuMotif(m);
if (!isDoctrinal && !isSeireishu) continue;
```

#### category フィルタの拡張

性霊集系：`category:無常 / 悲嘆 / 廻向 / 讃美`
教義書系：`category:密教教学 / 無常 / 景気導入 / 典故引用`

橋スロットに「教義的厳粛さ」を入れたい場合は `category:密教教学` を許可する必要がある。ただし全教義書 motif が橋に流入すると橋の文体バランスが崩れるので、サブフィルタとして：
- `文体:対句` OR `文体:短句` AND
- `一句性:核心` OR `category:無常`（無常含意のあるものを優先）

を併用するのが安全。

### 5.2 新 origin の追加

`BridgeCandidate.origin` 型に `'motif_doctrinal'` を追加（型ファイル更新必要）：

```ts
origin: 'daishi_kotoba' | 'motif' | 'motif_seireishu' | 'motif_doctrinal'
```

### 5.3 スコアリング設計

教義書系のスコアリング案：

```
base = 2
+ persona(故人:* マッチ): +2
+ category マッチ:
    category:密教教学: +1
    category:無常: +2
    category:景気導入: +1
+ 著作別重み（任意）:
    出典:般若心経秘鍵 / 即身成仏義 / 秘蔵宝鑰: +1（諷誦文との親和性高）
    出典:吽字義 / 大日経疏巻第一: +0（橋向きでない部分多）
+ preferred(hardness): +1
最大スコア 約 7
```

性霊集系（base 1）より base が高いのは「教義書の言葉は重みがある」ことを反映。daishi_kotoba 5 件（base 4-5）と同等〜やや下になる調整。

### 5.4 引用前置詞の動的判定

引き継ぎメモ（2026-05-07）の「注意：性霊集の文は『大師曰く』と引用するには文体が違う面があるため、『弘法大師のたまわく、』の前置詞をどう付けるかは要検討」に対応：

- `引用形式:大師曰く` タグ付き：「弘法大師のたまわく、」を前置
- `成句:famous`（漢文成句）：前置詞なしでそのまま朗誦
- 性霊集系（`引用形式:大師曰く` なし）：「大師の遺せる言葉に〜」または「弘法大師の御文に〜」を前置
- 菩提心論・大日経疏巻第一系：「大師の解する所に〜」または典故名で前置

詳細は kaimyo-app `fusonmon-renderer.ts` で実装。

### 5.5 観測ログの拡張

`source_counts` に `motif_doctrinal` を追加：

```ts
const source_counts = {
  daishi_kotoba: ...,
  motif_daishi: ...,
  motif_seireishu: ...,
  motif_doctrinal: ...,  // 新規
};
```

## 10. 残課題と検討事項

### 10.1 「引用形式:大師曰く」の境界例

弁顕密二教論などで空海が他経典を引用しつつ自説を述べる箇所（「楞伽法仏説法の文〜」等）の扱い。原則は「空海の地の文」のみ大師曰く付与、引用部分は典故タグのみ。

### 10.2 即身成仏頌の扱い

即身成仏義の即身成仏頌（8 句）は単独で成句化された有名偈頌。これを：
- (a) 8 motif として個別抽出
- (b) 1 motif として全頌をまとめて保持
- (c) 両方（個別 8 + 全体 1）

(a) と (c) は冗長性が出る。性霊集 motif 抽出原則に倣えば (a) が筋（1 motif = 1 句単位）。

### 10.3 大日経疏巻第二〜二十

大日経疏は全 20 巻。本書は巻第一のみを対象とするが、将来巻第二以降も取り込まれた場合の motif 抽出は同じルールを適用する。

### 10.4 他者著作（菩提心論・大日経疏）の motif の意義

「大師の言葉」とは厳密には言えないが、空海が読み解説した教義書として真言宗教学の中核を成す。`引用形式:大師曰く` は付かないが、`典故:菩提心論` `典故:大日経` の形で諷誦文の典故引用素材として有用。kaimyo-app 側で「経曰く」ルートとして別扱いになる。

### 10.5 既存 daishi_kotoba 5 件との整合

fusonmon.json 直書きの 5 件：
- 御遺告（1 件）
- 般若心経秘鍵（2 件）：「文殊の利剣諸戯を絶つ」「色は則ち空、空は則ち色」
- 即身成仏義（1 件）
- 吽字義（1 件）

段階2 で重複抽出される可能性がある（特に般若心経秘鍵）。orderer のテキスト一致チェック（fusonmon-orderer.ts 行 691-694）で除外される設計だが、テキスト末尾の句点や半角・全角差で漏れることがある。抽出時に既存 daishi_kotoba 5 件と機械的に照合してから motif 化する。
