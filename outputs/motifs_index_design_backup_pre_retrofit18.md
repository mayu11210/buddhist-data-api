# 文体モチーフ索引 設計メモ（2026-05-04 初版）

## 目的

buddhist-data-api を「**仏教知識の保管庫**」として位置付けし、保管されている各データに**多軸ハッシュタグ**を付与することで、kaimyo-app（諷誦文・戒名・法話生成アプリ）等の利用側がタグ検索で必要な素材を取り出せるようにする。

本メモは 2026-05-04 のケンシン Cowork 設計議論を畳み込み、以後セッションでの実装フェーズの設計指針を確定するために作成する。

---

## 1. 倉庫設計原則の再確認（2026-05-04 議論で更新）

### 大原則：buddhist-data-api は完全な倉庫である

- **倉庫はデータを保管し、タグを付ける**。それ以上のことはしない。
- **「このデータはアプリ X に使える／使えない」のような用途判断はしない**（キュレーション禁止）。
- **使えるかどうかは使う側が判断する**。倉庫は判断を肩代わりしない。

### 派生原則

- **アプリ固有の構造は倉庫に持ち込まない**：諷誦文の「初段／二段／三段／四段／五段」は kaimyo-app 固有の出力フォーマットであり、倉庫側のタグ・フィールドにしてはいけない。
- **アプリ用途で命名しない**：`fusonmon_*`・`kaimyo_*` のようなアプリ名空間を新規追加しない。データの自然な姿で命名する。
- **タグ軸を網羅的に整える**：倉庫の責務は「網羅性」と「タグの正確さ」にある。

### 役割分担（kaimyo-app 諷誦文生成を例に）

```
[kaimyo-app 諷誦文生成チーム]
 ┌─初段 注文担当者 ─┐  ┌─初段 受け取り担当者 → 初段 文書生成担当
 ├─二段 注文担当者 ─┤  ├─二段 受け取り担当者 → 二段 文書生成担当
 ├─橋  注文担当者 ─┼─→├─橋  受け取り担当者 → 橋  文書生成担当 ─→ 全体まとめ
 ├─三段 注文担当者 ─┤  ├─三段 受け取り担当者 → 三段 文書生成担当
 ├─四段 注文担当者 ─┤  ├─四段 受け取り担当者 → 四段 文書生成担当
 └─五段 注文担当者 ─┘  └─五段 受け取り担当者 → 五段 文書生成担当
        ↓ 各々が別々のタグ組合せで素材注文
[buddhist-data-api 倉庫]
 ・性霊集／秘蔵宝鑰／弁顕密二教論／菩提心論／三教指帰…
   の本文（書き下し+現代語訳+漢文）を多軸タグ付きで保管
 ・注文（タグクエリ）に応じて候補リストを返す
 ・「段」「硬さ UI」「故人タイプ UI」のような利用側固有の構造は持たない
```

「段」「初段／二段／…」の概念は **kaimyo-app 側に閉じる**。倉庫側は段を一切意識しない。

### 補注 B：代筆書状・代筆勅書の motif 化方針（2026-05-09 W1 マージで明文化）

性霊集には空海が他者の代筆として執筆した書状・勅書・表が多数含まれる：

| idx | 篇名 | 形式上の発信者 | 実際の代筆者 |
|---|---|---|---|
| 22 | 筆を奉献する表 | 嵯峨天皇宛・空海自身 | 空海 |
| 37 | 大使福州の観察使に与えるが為の書 | 遣唐大使藤原賀能 | 空海 |
| 88 | 玄賓法師に贈る勅書 | 嵯峨天皇 | 空海 |
| 105 | 泰範叡山の澄和尚に啓を答する為の書 | 弟子泰範 | 空海 |

これら代筆文書を motif 化する際の方針：

1. **形式上の発信者を `category:大師御言葉` で保持する**：「賀能曰く」「天皇曰く」相当の引用形式（`category:大師御言葉` または `引用形式:大師曰く`）を保持する。
2. **代筆の歴史的事実は `text_gendaigoyaku` の補注に明示する**：書き下し（`text_kakikudashi`）と雅文体訳（`text_gendai_gabun`）は形式上の発信者の声を保つが、学習者向け補注（`text_gendaigoyaku`）に「（空海が代筆）」「（藤原賀能名義・空海代筆）」等を併記する。
3. **故人タグは付与しない**：生者宛の代筆書状（観察使宛・節度使宛・天皇宛 等）には `故人:*` タグを付与しない（W1 完走時に確立した運用先例：idx=22／37／88／105／106／39 で踏襲）。
4. **主体軸は形式上の発信者 + 代筆者の双方を記録する**：本体既存の主体軸（例：`主体:泰範（代筆者：空海）`・idx=105 で導入）を踏襲する。新規導入時はマージセッションで合意する。
5. **対象軸は受信者を記録する**：書状の受信者（例：`対象:最澄`・idx=105・idx=106／`対象:福州観察使`・idx=37）を `対象:*` 軸で保持する。

### 補注 C：含意の射程 タグの「代表 motif 付与」原則（2026-05-09 確認）

`含意の射程` タグは当該篇内の **全 motif に一律付与しない**。以下のいずれかに該当する代表 motif にのみ付与する：

- **結句 motif**：篇の最終句で総括的な廻向・祈願・辞退表明を行う motif
- **発心・廻向の主体句 motif**：篇の動機を凝縮する代表句

これは含意の射程タグが「篇全体の意味的射程」を表すためで、篇内の景気導入・典故引用・人柄描写などの motif に重複付与すると、検索時のノイズとなる。kaimyo-app 等の利用側は本タグで篇単位の動機分類が可能になる。

W1 マージで付与した 3 motif（m378・m391・m404）はすべて該当篇の結句相当 motif である。

### 補注 D：戯曲形式の発言主体識別タグ運用（2026-05-11 retrofit 4 で確立）

三教指帰のように**戯曲形式（対話劇）**で進行する著作では、登場人物ごとに思想立場が異なるため、発言主体を識別するタグが必要となる。本体 W2 マージ後に retrofit 4 として、新規軸『発言者』を schema 0.2 に追加した（軸新設はマージセッションでの合意が必要・workshop_protocol §5 準拠）。

**新規軸：発言者**

| タグ値 | 該当人物 | 位置づけ |
|---|---|---|
| `発言者:空海` | 弘法大師空海ご自身 | 自序・地の文の編者 |
| `発言者:仮名乞児` | けみょうこつじ | 仏教代表・空海の分身（架空人物） |
| `発言者:亀毛先生` | きもうせんせい | 儒教代表（架空人物・「亀の毛」のように存在しないもの） |
| `発言者:虚亡隠士` | きょもういんじ | 道教代表（架空人物） |
| `発言者:蛭牙公子` | しつがこうし | 教化対象青年（架空人物） |
| `発言者:兎角公` | とかくこう | 蛭牙の伯父・儒教側人物（架空人物） |
| `発言者:或人` | あるひと | 仮名乞児への問者（無名） |

**運用ルール**：

1. **発言者軸は戯曲形式の motif に限定**：通常の論書・願文・書状等では発言主体が著者本人で固定的なため、発言者軸を付与しない（cluttered になる）。戯曲形式・対話形式の motif にのみ付与する。
2. **複数主体の motif には複数値を付与**：m666（虚亡隠士発言＋亀毛・蛭牙・兎角の合同改心）・m668（或人の問＋仮名乞児の答）・m675（亀毛公等廻心＋仮名乞児結語）のように、1 motif 内に複数発言者が登場する場合は、その全員を付与する。
3. **地の文・自序は『発言者:空海』**：m657（亀毛紹介の地の文）・m667（仮名乞児紹介の地の文）・m656（自序）等、ナレーター位置の motif は編者である空海を主体とする。
4. **category:大師御言葉・引用形式:大師曰く は仮名乞児・空海主体の motif のみ保持**：「大師御言葉」は弘法大師の御言葉を指すため、仮名乞児（空海の分身）と空海自身の発言にのみ付与する。亀毛先生・虚亡隠士・蛭牙公子・兎角公・或人の発言には付与しない（kaimyo-app 等で誤って引用される事故を防ぐ）。混在 motif（m668・m675）は仮名乞児が応答主体のため両者を保持する。
5. **将来の戯曲・対話形式著作への展開**：本軸は三教指帰に限らず、対話形式・問答形式で進行する他著作（経典の対話パート・論書の問答節等）にも転用可能。新規人物が登場する場合は本表に追加する（タグ値追加のみ・新規軸新設は不要）。

**retrofit 4 実施結果（2026-05-11）**：

- m656-m676 21 motif すべてに `発言者:*` を付与（+30 タグ）
- m662-m666 5 motif（虚亡隠士発言）から `category:大師御言葉`・`引用形式:大師曰く` を除去（-10 タグ）
- m668・m675 混在 motif は仮名乞児主体のため両者保持
- schema_version は 0.2 維持（フィールド構造変更なし・タグ値運用拡張のみ）

### 補注 E：成句連動の motif 識別タグ運用（2026-05-11 retrofit 5 で確立）

**新規軸：連動**

| タグ値 | 連動先 | 位置づけ |
|---|---|---|
| `連動:sg03` | sg03「父母所生身、速証大覚位」（即身成仏義出典の漢文成句） | 即身成仏義の最有名の決まり文句 |

### 雅文体訳の文体規定（2026-05-04 ケンシン採用・2026-05-05 倉庫側 schema 反映）

#### 採用に至った経緯

kaimyo-app 諷誦文 縦割り 初段スライス完走時（2026-05-04・kaimyo-app セッション）、ケンシンが gendaigoyaku 再構築版（出力 1）と kakikudashi 版（出力 2）を見比べて：

> 書き下しそのままではだめですね。
> でも大師の息遣いは聞こえてくる響きをもった文章ですね。
> 現代語でうまく表現できないものか。

3 段階の試訳（試案 A 雅文寄り／試案 B 中庸／試案 C 柔らか）を提示し、**試案 A（雅文寄り）を採用**。これが諷誦文本体として読み上げて意味が届きながら大師の息遣いも残る目標値となった。

#### 範例（m01 試案 A・確定済）

書き下し（kakikudashi）：
> 夫れ寥廓たる性虚は諸因を離れて而も凝然たり。飄蕩たる染海は衆縁に随て以て起滅す。

学習者向け説明訳（gendaigoyaku）：
> そもそも寥廓（りょうかく・広漠として広大な）たる性虚（しょうきょ・法性虚空・法身の体）は、諸因（一切の因縁）を離れて、しかも凝然（ぎょうねん・固く凝り立って動かぬさま）たる本体である。これに対し、飄蕩（ひょうとう・漂い揺らぐ）たる染海（ぜんかい・煩悩に染められた識海・八識心王の海）は、衆縁（多くの因縁）に随って起滅し変転する。

雅文体訳（gendai_gabun・試案 A）：
> そもそも、ひろやかなる法身の本体は、もろもろの因縁を超えて、しずかに動かぬものなり。
> されど、煩悩に染められたる心の海は、縁にしたがいて起こり、また消えゆく。

#### 文体規定（執筆ガイドライン）

1. **やまと言葉優先**：漢語（寥廓・諸因・凝然・飄蕩・染海・衆縁・起滅）はできる限りやまと言葉に開く（ひろやか・もろもろの因縁・しずかに動かぬ・煩悩に染められたる・縁にしたがいて起こり、また消えゆく）
2. **補注を埋め込まない**：括弧書きの読み・典故・字義解説（gendaigoyaku に含まれるもの）はすべて削ぎ落とす。「梵語 X」「『○○経』六喩」のような注釈は雅文体には入れない
3. **対句・畳語・呼びかけのリズムを保つ**：原文が対句なら対句で訳す。畳語（哀れなるかな、哀れなるかな…）はそのまま残す。呼びかけ（〜の子よ・〜の客よ）も残す
4. **雅文体の活用形**：「ものなり」「ぞ」「けり」「たまえ」「たり」「ゆく」「らむ」「なき」等の文語的活用を活かしつつ、文の組み立ては現代日本語の語順に近づける
5. **現代口語の混入を避ける**：「である」「変転する」「ものだ」「のようだ」「だが」などは使わない
6. **意味の取りこぼしは許容しても情緒の取りこぼしは避ける**：「性虚」を「法身の本体」と意訳的に開いてもよい（学術精度は gendaigoyaku が担う）
7. **儀礼朗誦に耐える長さ・呼吸**：1 文を読み下しやすい長さに区切る。原文が改行なしの長句でも、雅文体では適宜改行を入れて住職の朗読に資する

#### 既存 kakikudashi が既に雅文体形をしている場合

m06「哀れなるかな、哀れなるかな、哀れなる中の哀れなり。悲しきかな、悲しきかな、悲しき中の悲しみなり。」のように、書き下しが既にやまと言葉中心の純然たる雅文体になっている素材は、`text_gendai_gabun` を別途設けず**未設定（オプションだから空欄）**で問題ない。kaimyo-app 側のフォールバック規則「`text_gendai_gabun` が無ければ `text_kakikudashi` を使う」で同等の出力になる。

#### 成句（`成句:famous`）の扱い

「諸行無常」「色即是空」「父母所生身、速証大覚位」のような漢文成句は、儀礼朗誦の伝統で漢文のまま音読される。雅文体訳に開くと（「もろもろの行いは無常なり」等）儀礼上の重みを失うため、**`text_gendai_gabun` は設けない**（2026-05-05 ケンシン判断）。kaimyo-app 側は `成句:famous` タグを見て `text_kakikudashi`（漢文形）をそのまま諷誦文本体に流す。

将来「柔モード」で漢文成句を雅文体に開きたいニーズが出たら、その時点で個別に `text_gendai_gabun` を追加する余地を残す（フィールドはオプションだから後追加可能）。

### 書き下しルール（厳守）

- 倉庫の素材本文は **書き下し文を主**とする（`text_kakikudashi` 必須）
- 漢文（genten / 大正蔵原典等）は学術参照用にフィールドとして保持してよいが、**諷誦文生成・法話生成では使わない**
- ただし「諸行無常」「色即是空」「父母所生身」のような有名な漢文成句は、書き下しに変換せずそのまま使ってよい（`成句:famous` で識別）

### 「事例に見合う現代語表現」の役割分担

- buddhist-data-api 側 = neutral な雅文体訳と学習者向け説明訳を保管（事例非依存）
- kaimyo-app 側 = 故人プロファイルに応じて現代語表現を再構築（事例依存）
- 倉庫は「neutral 雅文体訳」「neutral 学習者向け説明訳」を返すのみ。事例適応は利用側の責任。

## 8. 残課題

### 設計フェーズ
- ファイル名・配置の最終決定（案 A〜D）
- 「硬さ」軸の倉庫原則上の妥当性再評価（kaimyo-app UI 由来のため、データ自然属性に分解する案：`漢語密度`・`対句密度`・`典故密度` 等）
- 「場面」軸の最小化（場面は用途寄りのため、データ自然属性として明らかな場合のみ付与）

### データフェーズ
- bodaishinron.json は kakikudashi のみで gendaigoyaku 未反映（候補 A 半完了状態）→ 反映完了後に菩提心論からの素材抽出可
- 御遺告未取込 → 橋プール拡張時に取込検討（要設計セッション）
- 経典本体（華厳経・法華経・大日経・理趣経・大智度論等）の取込検討 → 別途取込セッション必要
- 漢籍引用（論語・荘子・書経・詩経・史記等）の橋プール採否

### 実装フェーズ
- 検索 API の設計（既存 `/api/houwa/citations` 系列拡張 or 新エンドポイント）
- 索引ファイルのフォーマット最終確定（JSON / JSONL / SQLite）
- パフォーマンス（性霊集 500〜1000 素材 + 他著作 200〜500 素材で 1000〜2000 素材規模）
