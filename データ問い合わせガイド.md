# データ問い合わせガイド（NotebookLM モード）

このフォルダ `buddhist-data-api` のデータ**だけ**を情報源に、質問へ
**根拠（出典）つきで答える**ためのガイドです。NotebookLM のように
「手元のソースに基づいて質問に答える」窓口として、このチャットを使います。

> **使い方**：新しい Cowork チャットをこのフォルダで開き、最初に
> 「`データ問い合わせガイド.md` に従って、このフォルダのデータについて
> 質問に答えて」と伝えてください。以後、ここに書いた回答プロトコルで応答します。
> （このチャットをそのまま続けて質問してもらっても構いません。）

---

## 1. このチャットの役割

**やること**

- 蓄積済みデータ（空海の著作本文・書き下し・現代語訳・原典・横断索引・
  密教大辞典・モチーフ集）を検索・参照し、質問に答える。
- 答えには必ず**出典**（ファイル名＋著作名＋篇名／idx／段落番号）を付ける。
- 原文（書き下し）と現代語訳の両方を、求めに応じて引用する。

**やらないこと**

- データの**新規取り込み・編集・翻訳作業はしない**（それは取込作業用の
  `CLAUDE.md` 側のフロー。混同しない）。
- コーパスに無い事柄を推測で補わない。無ければ「このコーパスには無い」と明示し、
  必要なら一般知識である旨を断ってから補足する。
- 原本ファイル（`data/` 配下）は**改変しない**（読むだけ）。

### 2-2. 密教大辞典・横断索引 … `data/mikkyou/`

- **密教大辞典**：かな行ごとに分割（`a.json` `i.json` … `wa.json`、`si1〜4` `ko1〜3` など）。
  各ファイルは `{ section: "あ", source: "密教大辞典", entries: … }`。用語の意味・定義を引くならここ。
- **横断索引（7カテゴリ）**：著作ごと・全体の `index_*.json`。
  - `terms`（密教教学用語）／`citations`（典拠・引用文献）／`kukai_works`（空海著作への言及）／
    `sanskrit`（梵語）／`kaimyo_jukugo`（戒名向け熟語）／`persons`（人物）／`places`（地名）
  - 例：性霊集なら `index_shoryoshu_terms.json` `index_shoryoshu_persons.json` …
  - 各著作なら `index_<著作名>_<カテゴリ>.json`
- 特殊データ：`sanskrit.json`（梵語）／`nenpyo*.json`（年表）／`horyu_keifu.json`（法流系譜）／
  `mikkyou_inzu.json`（印図）／`index_butsuzo.json`（仏像）／`index_kanji.json`（漢字）／
  `ryakuji.json`（略字）／`supplement.json`（補遺）。

### 2-3. モチーフ集 … `data/indices/motifs.json`

弔辞・追悼・願文などから抽出した名句モチーフ（id `m001`〜）。各 motif は
`{ id, source:{corpus, idx, 巻番号, 篇名}, text_kakikudashi, text_gendai_gabun,
text_gendaigoyaku, tags:[…] }`。`tags` は多軸（`category:` `主題:` `典故:` `場面:`
`一句性:` `文体:` `出典:`）。「無常を詠む句」「慈悲のテーマ」などテーマ検索に使う。

### 2-4. テンプレート … `data/templates/`

表白・諷誦文などの定型（`hyobyaku.json` `fusonmon.json`）。

## 3. データ構造 早見表

**dict 型**（多くの著作）: `section`（著作名）, `source`, `description`,
`kakikudashi`（書き下し）, `gendaigoyaku`（現代語訳）, `genten`（漢文原典）,
`genten_source`（原典URL）。著作固有の専門サブセクションが付くものもある
（例：吽字義の `ji_so` `ji_gi`、弁顕密二教論の `sanshin_setsu` 等）。

**list 型**（性霊集 `shoryoshu_miyasaka.json`）: 112 要素の配列。各篇は
`{ 巻番号, 篇名, 原文題, 詩形, 総字数, ページ:[{ kakikudashi, gendaigoyaku }] }`。
**篇は idx 0〜111 で参照**（巻第一〜巻第十）。

**dict_paragraphs 型**（注釈書・開題類）: `paragraphs:[{ section, kakikudashi,
gendaigoyaku }]` の段落単位。`outline`（科段）と `base_text_ref` を持つ。

**索引型**（`index_*.json`）: `{ schema_version, category, source_corpus, summary,
entries:[{ term, definition, occurrence_count, … }] }`。統計は `summary` から取る。

## 4. 回答プロトコル（NotebookLM の肝）

1. **まず目録と索引を見る** → `_corpus_manifest.json` で対象著作を特定し、
   `index_*` で語・人物・典拠の所在を絞ってから、本文ファイルの該当箇所を開く。
   横断的な語の検索は全文 grep も併用する。
2. **答えは必ずデータに基づける。** 推測で埋めない。
3. **出典を明示する。** 形式の目安：
   `〔出典：性霊集 idx77「仏経を講演して四恩の徳を報ずる表白」（第八巻）／shoryoshu_miyasaka.json〕`
4. **原文と訳を区別して引く。** 書き下し＝原資料、現代語訳＝既訳。混ぜない。
5. **コーパスに無いときは正直に。**「このコーパスには該当が見当たりません」と述べ、
   一般知識で補う場合はその旨を断る。
6. 数値・頻度・分布は索引の `summary` / `occurrence_count` / `篇分布` を根拠にする。

---

## 5. 質問タイプ別レシピ（例）

- **「〈語〉はどの著作・どこに出る？」** → 該当 `index_*_terms.json`（または全文 grep）で
  occurrence と篇/idx を特定 → 本文を引用。
- **「〈篇/著作〉の現代語訳を見たい」** → 性霊集は `shoryoshu_miyasaka.json` の idx、
  その他は各 `*.json` の `gendaigoyaku`。
- **「人物〈X〉への言及は？」** → `index_*_persons.json`。地名は `places`、典拠は `citations`。
- **「〈テーマ〉を詠む名句は？」**（無常・慈悲・菩提心など）→ `motifs.json` を
  `tags` の `category:`／`主題:` で絞る。
- **「〈用語〉の意味は？」** → `data/mikkyou/` の密教大辞典（かな行ファイル）＋ `terms` 索引。
- **「梵語〈…〉は？」** → `sanskrit.json` ／各 `index_*_sanskrit.json`。

---

## 6. 注意

- ルートの `CLAUDE.md` は**データ取り込み・翻訳作業用**の運用メモ（巨大）。
  この問い合わせモードとは目的が違うので混同しない。
- `data/` 配下の原本は**読むだけ**。質問対応で書き換えない。
- 「最新の状態」を答えるときは、記憶ではなく必ず `_corpus_manifest.json` 等の
  実ファイルを開いて確認する。
